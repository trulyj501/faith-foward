# 콘텐츠 스타일 가이드

이 문서는 **Faith Forward** 플랫폼의 상세 페이지에서 Markdown이 어떻게 렌더링되는지 보여주는 스타일 가이드입니다. 글을 작성하실 때 이 가이드를 참고하여 시각적으로 일관되고 아름다운 콘텐츠를 구성해 보세요.

## 1. 헤더 (Headings)

헤더는 콘텐츠의 구조를 잡는 데 사용됩니다. `#`의 개수에 따라 수준이 결정됩니다.

### H3 헤더 (세부 주제)
주제 내부의 작은 단위를 나눌 때 사용합니다.

#### H4 헤더 (최소 단위)
컴포넌트나 아주 작은 항목을 설명할 때 적합합니다.

---

## 2. 텍스트 스타일 (Typography)

일반적인 본문 텍스트는 가독성을 최우선으로 합니다.
**굵은 글씨(Bold)**는 강조할 때, *기울임꼴(Italic)*은 인용이나 특정 용어를 부각할 때 사용합니다. `코드 스타일`은 인라인 코드나 키워드를 표현할 때 좋습니다.

> **인용문 (Blockquote)**
> 신앙과 기술의 교차점에서 우리는 더 나은 미래를 봅니다. 이 스타일은 중요한 문장이나 외부 인용구를 강조할 때 사용하세요.

### 이스케이프 대시 (Escaped Dash)

`- text`는 리스트(bullet)로 인식되지만, `\-`를 쓰면 리스트 스타일 없이 `-` 기호 그대로 출력됩니다.

| 입력 | 출력 |
|------|------|
| `- 항목` | 리스트 불릿(•) |
| `\- 항목` | \- 항목 (기호 그대로) |

인용문 안에서도 동일하게 사용할 수 있습니다.

> \- 이것은 리스트가 아닌 대시 텍스트입니다.
> \- 인용문 안에서도 동일하게 작동합니다.

### 콜아웃 박스 (Callout Box)
`> [!NOTE]`, `> [!TIP]`, `> [!WARNING]`, `> [!IMPORTANT]`, `> [!INFO]` 문법으로 강조 박스를 만들 수 있습니다.

> [!NOTE] 이것은 NOTE 콜아웃입니다. 배경이 파란색 계열로 표시됩니다.

> [!TIP] 이것은 TIP 콜아웃입니다. 에메랄드 배경으로 유용한 팁을 강조할 때 사용하세요.

> [!WARNING] 이것은 WARNING 콜아웃입니다. 주의가 필요한 내용을 노란색 계열로 강조합니다.

> [!IMPORTANT] 이것은 IMPORTANT 콜아웃입니다. 중요한 정보를 보라색 계열로 표시합니다.

> [!INFO] 이것은 INFO 콜아웃입니다. 추가 정보나 참고 사항을 하늘색으로 표시합니다.

## 3. 리스트 (Lists)

### 순서 없는 리스트
- 기획 (Planning)
- 디자인 (Design)
- 개발 (Development)
- 프론트엔드
- 백엔드

### 순서 있는 리스트
1. 첫 번째 단계: 주제 선정
2. 두 번째 단계: 자료 조사
3. 세 번째 단계: 초안 작성

---

## 4. 코드 블록 (Code Blocks)

기술적인 내용을 공유할 때는 언어를 명시하여 코드 블록을 작성할 수 있습니다.

```typescript
function calculateInsight(data: string[]): number {
  console.log("분석을 시작합니다...");
  return data.length * 100;
}
```
